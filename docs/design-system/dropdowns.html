Dropdowns

Select elements cannot be styled, but this component uses a workaround. The select itself is hidden, and styled HTML is injected into an empty div that comes after the select. Placing a class "select" calls the component. Inside the "select" div there is a label, the hidden select element, and the empty div. Even though the select is hidden, the empty div reads all the metadata built into the select, so the markup would contain all the options and values as usual.

Initializing all dropdowns

<script>
  include(dropdown());
</script>

Normal Select 

<!-- children order matters -->
<div class="select">
  <label for="dropdown">Label</label>
  <select id="dropdown">
    <option value="" selected>Choose</option>
    <option value="option 1">Option 1</option>
    <option value="option 2">Option 2</option>
    <option value="option 3">Option 3</option>
  </select>
  <div></div>
</div>

Multiple Select

Multiple select allows multiple values to be accepted in a dropdown.

<!-- children order matters -->
<div class="select multiple">
  <label for="multiple-dropdown">Label</label>
  <select id="multiple-dropdown" multiple>
    <option value="" selected>Choose</option>
    <option value="option 1">Option 1</option>
    <option value="option 2">Option 2</option>
    <option value="option 3">Option 3</option>
  </select>
  <div></div>
</div>

Header Select

This is a special select dropdown that is found in the header of the Document Management System. It does not require JS, so it doesn't need to be initialized. The component has a visible container that reveals a hidden dropdown on hover.

<div class="header-select">
  <div class="container">
    <div class="ha-symbol color"></div>
    <h3>Document Management System</h3>
    <div class="line"></div>
  </div>
  <div class="hidden-container">
    <p class="caption">See Broker Portal View</p>
  </div>
</div>
